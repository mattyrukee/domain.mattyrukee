Mattyrukee tech on Initia Testnet Snapshot Server Setup

The snapshot is tailored for node operators aiming to efficiently run a node on the Initia chain. To minimize its size while maintaining its effectiveness as a validator, we employ specific settings to conserve disk space. It's advisable to sync with our snapshot regularly because Tendermint chain storage expands over time, despite pruning efforts. As we periodically state-sync our snapshot nodes, you may observe occasional instances where the snapshot size is remarkably small, streamlining its integration into your database for space efficiency and simplified setup. We conduct node snapshots daily for the testnet.

Initia Testnet Node Guide

Stop your node by running the following command:

sudo service initia stop

Reset your node to erase the node database. Make sure to back up your priv_validator_key.json before resetting.

WARNING: If you're using the snapshot on a validator node during a chain halt, ensure you back up priv_validator_state.json and replace it after the snapshot is extracted but before starting the node process.

Back up priv_validator_state.json if needed 
cp ~/.initia/data/priv_validator_state.json  ~/.initia/priv_validator_state.json
Reset your node. This will erase your node database 
# On some tendermint chains  

initiad unsafe-reset-all


# On other tendermint chains

initiad tendermint unsafe-reset-all --home $HOME/.initia --keep-addr-book
Stream and Decompress the snapshot to your database location. You database location will be something to the effect of ~/.initia depending on your node implemention. 
curl -o - -L http://dev.mattyrukee.xyz/snapshots/snapshot_height_237843.tar.lz4 | lz4 -c -d - | tar -x -C $HOME/.initia
Replace with the backed-up priv_validator_state.json 
cp ~/.initia/priv_validator_state.json  ~/.initia/data/priv_validator_state.json
Restart your node 
sudo service initia start
Make sure that your node is running 
sudo service initia status
sudo journalctl -u initia -f
